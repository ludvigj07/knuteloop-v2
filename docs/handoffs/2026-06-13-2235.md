## Gotchas / things to remember

- **Deploy-time invariant (do NOT forget before any non-local deploy):** set `NODE_ENV=production` **and** a real `JWT_DEV_SECRET`. The config guard now refuses the committed default in prod, but the residual fail-open is `NODE_ENV` *unset* → it silently defaults to `development` (which mounts the dev token endpoint). When wiring the real deploy, add an explicit `ENABLE_DEV_ROUTES` flag + a committed `.env.example` as belt-and-suspenders.
- **The whole app runs on placeholder HS256 dev auth.** `middleware/auth.ts` verifies dev tokens in all envs, trusts `school_id`/`role` from the token with no DB lookup and no `token_version` check (TODO in code). This is **deliberately deferred to the Vipps auth epic** — the real fix is RS256/JWKS + DB-backed user lookup. Don't deploy to prod until that lands.
- **Local Postgres at `localhost:5432` (`postgres:postgres`) is required** to run the api test suite (incl. the new `dev.test.ts`). It was up this session; CI provides it via a `postgres:17` service. NOT testcontainers.
- **`gh` is not on the bash PATH** — it lives at `C:\Program Files\GitHub CLI\gh.exe`. Authed as `ludvigj07`.
- **`pnpm dev` does not set `NODE_ENV`** — it comes from `.env`/dotenv, else defaults to `development`. That's why removing the default would break the local loop (so it was kept).
- **Re-running `pnpm dev:setup` WIPES the dev DB** and rewrites the Loke token in `apps/mobile/.env`. That's intended (predictable start state).
- The dev-login screen shows **no "✓ aktiv" highlight on first load** (you're on the env-token Loke but `activeUser` is null until you tap) — the new banner explains this; it's expected, not a bug.
