## What got done today

Built the **knutebibliotek** (knute library) as production RN, in two slices. Started from a `/knuteloop-design` review: the design kit placed Bibliotek + Knuteboka as global bottom tabs, but it's a **knutesjef-only** tool — production already had it right under `admin/`, so everything stays under `admin/` (reached from the Knutesjef panel), never the main tab bar.

**PR 1 — `feat/library-browse-polish` (browse + import polish):**
- New reusable `Sheet` primitive (`components/primitives/Sheet.tsx`) — bottom sheet via RN Modal portal + Reanimated slide, reduced-motion aware. (PR 2 uses it too.)
- `admin/bibliotek.tsx` rebuilt to the kit's look: catalog rows with folder tile, points/difficulty chips, **18+ / Tekst** flags, tap-to-open **detail sheet** (`KnuteDetailSheet`).
- **18+ friction-gate**: importing an adult knute routes through a confirm sheet (ADR-0015). Both the row quick-add and the sheet go through one gate.
- Shared `components/library/libraryMeta.tsx` + behavioural test `LibraryKnuteRow.test.tsx`.

**PR 2 — `feat/library-knuteboka` (folder management):**
- Folders API client added to `lib/api.ts` (list w/ counts, create, delete, add/remove membership, `fetchKnuterByFolder`) — wires the **existing** `/api/folders` + `/api/knuter?folderId=` endpoints. **No backend changes.**
- `app/admin/knuteboka.tsx` — folder overview: folders w/ counts, create folder, link to library.
- `app/admin/folder/[id].tsx` — folder detail: knuter in folder, remove, add-knute picker, delete folder.
- `components/library/folderUi.tsx` (FolderRow, AdminKnuteRow, AddKnutePickerSheet) + test.
- Knutesjef panel (`admin/index.tsx`) gained a **"Knuteboka"** row.

**Decision settled:** the library's filter pills are theme **tags** (`suggestedFolder`); Knuteboka manages the **real per-school folders** table. Two distinct concepts, kept separate (the design kit conflated them).

**Two honest deviations from the kit:** no "X skoler har denne" social proof (API exposes no school count — would be fake); folder rename/reorder, an "Alle knuter" view, and creating a brand-new custom knute directly into a folder were **deferred** (good next PR).

## Running it on the laptop (the .env files are NOT in git)

```bash
# 1. Postgres in Docker (throwaway local dev DB)
docker run -d --name knuteloop-pg \
  -e POSTGRES_PASSWORD=postgres -e POSTGRES_USER=postgres -e POSTGRES_DB=postgres \
  -p 5432:5432 postgres:17-alpine

# 2. Create apps/api/.env  (only DATABASE_URL is required; dev secret/storage default safely)
#    NODE_ENV=development
#    DATABASE_URL=postgres://postgres:postgres@localhost:5432/knuteloop_dev

# 3. Seed + run (dev:setup wipes/recreates the local dev DB and writes a fresh
#    Loke knutesjef token into apps/mobile/.env automatically)
pnpm dev:all
```

Then open the app → **Knutesjef tab → Bibliotek / Knuteboka**. (`pnpm mobile` → press `w` for web or scan the QR for native.) Reminder: start the Postgres container **before** `pnpm dev:all`, or it fails with `ECONNREFUSED 5432`.
