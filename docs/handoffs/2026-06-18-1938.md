## Gotchas / things to remember

- **`tenantContext` COMMITs on a thrown HTTPException** (latent, project-wide). Hono catches handler errors per-frame and runs onError without re-throwing, so the transaction wrapper commits even on a thrown error. **Safe today ONLY because every handler throws BEFORE its first write** — the import service documents this load-bearing ordering. Durable fix: `if (c.error) throw c.error` after `await next()` in tenant-context, tested across all endpoints. Tracked in `project_premerge-review-followups` memory #3.
- **Read-only library tables depend on app_role NOT owning them** — library_* have no RLS, so a table OWNER bypasses the REVOKE. Production migrations MUST run as a privileged role (superuser/admin_role), never as app_role. Documented in migration `0014` + `schema/library.ts`.
- **Hand-written migrations need a snapshot with a fresh `id` + `prevId` pointing at the previous snapshot** (not a verbatim copy) or `drizzle-kit generate` chains break. `0014_snapshot.json` was fixed this way.
- **`category` enum names ≠ folder names.** Import maps `suggested_folder` → category via `CATEGORY_BY_FOLDER` in `lib/library-import.ts` (Dobbel→Dobbelknuter, Rampestrek→Fordervett-knuter, etc.).
- A stray empty `src/db/migrations/meta/_journal.json` got created at the repo ROOT by a misdirected `drizzle-kit` run — deleted. Watch for it if you run drizzle-kit from the wrong CWD.
